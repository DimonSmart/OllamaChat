# IDD-0020.adr-csharp-dotnet-and-microsoft-agent-framework

## Status

Accepted

## Context

OllamaChat is an educational project for creating and using modern agent technologies. Its implementation should remain aligned with the modern Microsoft agent ecosystem.

## Decision

Use C# and .NET as the product implementation platform. Build agent and workflow capabilities on the current stable features of Microsoft Agent Framework and its related Microsoft AI abstractions, adopting newer applicable stable capabilities as the framework evolves.

## Alternatives Considered

- A language-agnostic implementation approach was not selected because C# and .NET are intentional learning and implementation constraints.
- Freezing the project to a specific library version was not selected because learning current framework capabilities is a project goal.

## Consequences

Future work evaluates Microsoft Agent Framework capabilities before introducing parallel agent-runtime abstractions. Upgrades may require behavior verification, but dependency versions remain implementation-managed rather than intent-managed.
