# IDD-0010.spec-application-settings

## Intent

Let users set application-wide defaults for chat, participant naming, voice input, and retrieval behavior.

## Related Specifications

Related: IDD-0001, IDD-0004, IDD-0005.

## Behavior

Users choose chat-history processing, default messages, participant display names, voice-input enablement and model, and embedding chunking settings. Voice transcription accepts an audio upload and returns text or a user-facing error.

## Durable Architecture And Constraints

Settings act as defaults for agent templates that do not provide an override.

## Non-Goals

This surface does not edit individual agent instructions.

## Acceptance Criteria

- Settings persist and affect subsequent applicable sessions.
- Voice input can be disabled without affecting text chat.
- Invalid or oversized audio receives a clear error response.

## Verification

Automated coverage verifies settings persistence, history modes, and voice-input validation.
