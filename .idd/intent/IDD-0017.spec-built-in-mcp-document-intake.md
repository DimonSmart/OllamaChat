# IDD-0017.spec-built-in-mcp-document-intake

## Intent

Provide an opt-in built-in MCP server that reads Markdown-compatible files and normalizes Markdown content for agent use.

## Related Specifications

Related: IDD-0007, IDD-0008.

## Behavior

The server reads an eligible local Markdown, text, or Markdown-extension file and returns normalized Markdown with metadata. It can also normalize supplied Markdown content and derive fallback title and document statistics.

## Durable Architecture And Constraints

Read operations do not modify the source file.

## Non-Goals

The server does not convert arbitrary binary office documents.

## Acceptance Criteria

- Eligible files return normalized content and metadata.
- A supplied fallback title is used when a document lacks a heading.

## Verification

Automated coverage verifies supported-file validation, normalization, title selection, and metadata.
